RU who
RUwho.icu

RUwho is a service that allows for easier searching of the public Rutgers University people database. RUwho is unaffiliated with Rutgers.

Designed to expand upon the official search platform; it's advantageous by returning results in real-time as a query is being entered alongside returning contact info of each match without the need to open a new tab for each individual.

A Flask backend integrates with a FastAPI wrapper to fetch from the Rutgers DB. This project furthered my experience with Flask, HTML, JS, CSS (including media quieries), and CORS. This was my first time using FastAPI & SlowAPI, Waitress, Intersection Observers, New Relic (NRQL), and HTTPX (over Requests) and my first time using AJAX, localStorage, asyncio, and rate limiting in production.

StockScope
StockScope.net

StockScope is a real-time press release aggregator for listed companies publishing on prominent newswires.

StockScope gathers releases from public newswire APIs, filters and categorizes them, and displays them in a central feed with the goal to provide investors a trading advantage without needing to constantly refresh multiple newsrooms.

A Sanic backend displays data from a MongoDB database. This project expanded my experience with MongoDB (pymongo), front-end web dev (HTML/JS/CSS), scheduling, and asyncio. This was my first time using Sanic and websockets and my first time using webhooks, Google Analytics, sessions, and an async framework in production.

Initially intended just to aggregate PRs, I saw an opportunity to shift StockScope's abilities to algorithmic trading following ChatGPT's surge. This project now autonomously trades American equities by running sentiment analysis on PRs in real time and making predictions based on recent market and historical data. I've since implemented multiple LLMS, including GPT-3.5/4, LLaMA, Qwen, PaLM, and Mistral models. This expansion familiarized me with backtesting (on countless months of high-quality logged data) and the Alpaca brokerage API.

After months of paper trading to optimize and hone my prompt engineering and algos, the StockScope backend has generated consistent weekly returns outperforming the S&P and Nasdaq.

bricks.deals
bricks.deals

bricks.deals is a Lego deal aggregator among leading US retailers. Sets are filtered to show only below-retail-price and in-stock deals available for purchase online. Expired deals and deal durations are visible as well.

Deals can be sorted by various criteria and filtered by store and theme. This lightweight web app excludes the clutter present on these stores' individual sites in a concise and neat manner.

A Flask backend is used to fetch data from a MongoDB database while a scheduling container periodically updates it. This weekend project advanced my experience with Flask, MongoDB, frontend web development, and manipulating API data. This was my first time using bs4 in production and MX records.
